# Upload Audio Files to Google Drive

Upload audiobook files generated by `/geno-media-audiobooks-create` or `/geno-media-audiobooks-recurse` to a structured folder in Google Drive.

## Input

`$ARGUMENTS` — Optional. Can be:
- A folder path to scan for audio files (absolute or relative to the current working directory)
- `show` to display current audio upload config
- Empty to scan the current working directory recursively

## Workflow

### 1. Load config

Read the config file at `~/.geno-tools/geno-media/audio-upload/config.json`. If it doesn't exist, create it with these defaults:

```json
{
  "google_account": "42euge@gmail.com",
  "drive_mount_base": "~/Library/CloudStorage",
  "default_folder": "Audiobooks"
}
```

If `$ARGUMENTS` is `show`, display the current config and stop.

Resolve the full Google Drive mount path: `<drive_mount_base>/GoogleDrive-<google_account>/My Drive`

Verify the mount exists. If not, tell the user their Google Drive may not be mounted (they need Google Drive for Desktop signed into `42euge@gmail.com`).

### 2. Discover audio files

Resolve the scan path: if `$ARGUMENTS` is non-empty (and not `show`), use it. Otherwise use the current working directory. Call it `ROOT`.

Recursively find all `.wav` and `.mp3` audio files under `ROOT` using Glob (`**/*.wav`, `**/*.mp3`).

**Prioritize files with metadata:** For each audio file, check if a corresponding `audiobook_meta.yaml` exists in the same directory. If it does, read it for duration/voice info. If it doesn't, still include the file — many audiobook outputs won't have metadata.

**Skip:**
- Any path containing `.venv/`, `node_modules/`, `__pycache__/`, `.git/`
- Files smaller than 10KB (likely empty/corrupt)
- Obvious non-audiobook files (e.g., system sounds, notification sounds) — use your judgment based on filename and location

### 3. Read metadata and build upload plan

For each qualifying audio file:
1. Check if `audiobook_meta.yaml` exists in the same directory. If it does, read it and extract: `audio_file`, `voice`, `speed`, `duration`, `chapters` (if present)
2. If no metadata exists, get the file size and estimate duration from file size (WAV at 24kHz mono ~ 48KB/sec, MP3 at 128kbps ~ 16KB/sec)
3. Determine the **relative path** from `ROOT` to the audio file's parent directory — this becomes the subfolder structure in Google Drive

Build the upload plan showing the mapping:

```
Source -> Destination (in Google Drive)

<ROOT>/docs/paper1/Paper Title.wav
  -> My Drive/Audiobooks/docs/paper1/Paper Title.wav

<ROOT>/research/notes/Deep Learning.wav
  -> My Drive/Audiobooks/research/notes/Deep Learning.wav
```

The destination structure is: `My Drive/<default_folder>/<relative-path-from-ROOT>/<audio-filename>`

### 4. Present the plan

Show the user:
- Number of audio files found
- Total size of all files
- Total audio duration (sum of durations from metadata)
- The full upload mapping (source to destination)
- The Google account being used

Ask the user to confirm before proceeding. They may:
- Exclude specific files
- Change the destination base folder
- Choose to also upload the `audiobook_meta.yaml` files alongside the audio (default: yes)
- Choose to skip MP3 conversion and upload WAVs as-is (default: convert to MP3)

### 5. Convert WAV files to MP3

By default, **always convert `.wav` files to `.mp3` before uploading** to save Drive storage (~80% smaller). Files that are already `.mp3` are uploaded as-is.

For each `.wav` file in the plan:
```bash
ffmpeg -y -i "/path/to/file.wav" -codec:a libmp3lame -qscale:a 2 "/path/to/file.mp3"
```

**Important:** Use properly quoted paths — filenames often contain parentheses and spaces. Always wrap paths in double quotes.

Use the `.mp3` files for upload instead of `.wav`. Do NOT delete the original `.wav` files from the source.

If the user explicitly asks to skip conversion, upload the `.wav` files directly.

### 6. Upload files

For each file in the plan:

1. Create the destination directory structure if it doesn't exist:
   ```bash
   mkdir -p "<drive_mount>/My Drive/<default_folder>/<relative-path>"
   ```

2. Copy the audio file:
   ```bash
   cp "<source-audio-file>" "<drive_mount>/My Drive/<default_folder>/<relative-path>/"
   ```

3. Copy the metadata file (if user opted to include it):
   ```bash
   cp "<source-dir>/audiobook_meta.yaml" "<drive_mount>/My Drive/<default_folder>/<relative-path>/"
   ```

4. If a file with the same name already exists at the destination, ask the user whether to:
   - Overwrite
   - Skip
   - Rename (append timestamp)
   - Apply choice to all remaining conflicts

Report progress after each file: `[3/12] done docs/paper1/Paper Title.mp3 (4.5 MB)`

### 7. Summary

After all files are uploaded, show:

```
Audio upload complete!

Uploaded: 10/12 files
Skipped:  2 (already existed)
Failed:   0

Total size uploaded: 1.2 GB
Total audio duration: 3h 45m 12s

Destination: Google Drive (42euge@gmail.com)
Folder: My Drive/Audiobooks/

Files:
  Audiobooks/docs/paper1/Paper Title.mp3           (12:34, 4.5 MB)
  Audiobooks/research/notes/Deep Learning.mp3       (8:21, 3.1 MB)
  ...
```

### 8. Verify

For each uploaded file, verify:
- The file exists at the destination path
- The file size matches the source
- Report any mismatches

## Important Notes

- This skill uses **local Google Drive mount** (Google Drive for Desktop), not the API — files sync automatically once copied
- Large files may take time to sync to the cloud even after the copy completes
- The structured folder hierarchy mirrors the source project structure, making it easy to find files
- All `.wav` and `.mp3` files are discovered; those with `audiobook_meta.yaml` get richer metadata in the upload plan
- WAV files are always converted to MP3 by default (~80% smaller) before uploading. Only upload WAVs if the user explicitly asks to skip conversion
